# Visual System

Phase Three keeps the existing dark red, ink, cream, muted gold, paper, stamp, dossier, and constructivist language while assigning each major system a clearer visual role.

## Cinematic

The six-scene introduction now moves from geographic collapse, through rail/front/famine civil-war pressure, competing papers, a congress resolution, an explicit Workers' Opposition meeting, and a map-connected title. The sequence is skippable, replayable, query-addressable for review, captioned, and replaced with a static faction-focused version under reduced motion.

The congress frame uses a clearly labeled original artistic reconstruction. Leader records use designed silhouette dossiers with initials and an explicit no-photograph claim.

## Persistent identity

Setup and the campaign sidebar repeat the original faction symbol, Workers' Opposition name, leaders, banned legal position, player background, selected opening strategy, institutional base, strengths, weaknesses, and dilemma. The interface does not imply that the player is Lenin or controls the Soviet state.

## Information hierarchy

- The top bar promotes stability, worker support, and exposure; other resources move into an expandable drawer.
- Economy opens with actual monthly campaign-history series and a faction profile; the numeric ledger is secondary.
- Regions compare food, unrest, unions, security, and faction reach against neighbors.
- Institutions use an attitude/influence relationship graph before detailed records.
- The named vote uses a seating diagram before delegate dossiers.
- Characters use designed dossier portraits, office badges, agendas, and status.
- Newspapers use publication-specific mastheads, columns, security stamps, paper color, and suppression treatment.

Charts are SVG built from saved simulation values. Accessible tables or text descriptions preserve the same information without relying on shape or color alone.

## Motion and accessibility

Map and cinematic animation obey preferences and `prefers-reduced-motion`. Colorblind patterns, visible focus, semantic headings/controls, text scaling, captions, and keyboard map access remain intact. Presentation preferences do not modify simulation state.
